# DropDrop — Context

Glossary of domain terms. No implementation detail here — see `docs/adr/` for decisions and rationale.

## Terms

**Box**
A shareable file storage unit. Backed 1:1 by a directory on the server filesystem (`./files/<boxId>/`). Created empty via `POST /api/box/create`.

**Box ID (slug)**
The identifier and sole access credential for a Box. A 4-word passphrase (e.g. `correct-horse-battery-staple`-style, generated by `secure-password-utilities`), not a UUID. Doubles as the secret in the share URL (`/box/<boxId>`). See [[0001-slug-based-access-control]].

**Node**
Base shape shared by every entry in a Box: `id`, `type` (`dir` | `file`), `boxId`, `path` (array of path segments), `name`.

**Directory**
A `Node` of type `dir`. Holds a list of child `File`/`Directory` entries and an optional rendered `readme`.

**File**
A `Node` of type `file`. Carries `mime`, `size`, and optionally `base64` content.

**Recent Box**
Client-side only concept — the last 5 Box IDs a browser has visited, kept in `localStorage` (never sent to server, not a server-side record).

**Readme**
A `README.md` file at the root of a Directory, rendered to sanitized HTML and shown below the file table for that Directory.
